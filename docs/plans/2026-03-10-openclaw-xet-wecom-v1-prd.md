# OpenClaw x 小鹅通 x 企业微信 V1 PRD

**版本**: V1.0
**日期**: 2026-03-10
**负责人**: 增长运营 / 直播运营

---

## 1. 背景与目标

当前直播运营存在三类重复工作:
1. 在小鹅通后台手工创建直播与运营配置。
2. 手工整理直播链接、直播带货信息、优惠券信息。
3. 在企业微信分渠道重复发送对外触达消息。

V1 目标是把以上流程收敛为一条可执行任务，优先覆盖对外客户触达。

## 2. 目标用户与场景

目标用户:
1. 直播运营同学
2. 增长运营同学
3. 活动负责人

核心场景:
1. 手动触发一次活动链路（创建直播 -> 生成素材 -> 群发）。
2. 定时触发开播提醒。
3. 同步触达两类对象: 客户群（优先）与外部联系人。

## 3. V1 范围

### 3.1 In Scope（现阶段）

1. 小鹅通网页自动化创建直播。
2. 添加直播带货商品。
3. 添加优惠券。
4. 生成群发素材（直播链接 + 运营信息文案）。
5. 企业微信对外群发:
   1. 客户群群发（P0）
   2. 外部联系人群发（P1）
6. 发送主体绑定真实成员 `userid`。
7. 支持手动触发与定时触发。
8. 支持任务状态查询和失败重试。

### 3.2 Out of Scope（V1 不做）

1. 小鹅通开放平台 API 通道。
2. 企业微信内部应用消息。
3. 企业微信群机器人推送。
4. 智能人群分层与 A/B 自动优化。

## 4. 功能需求

### 4.1 活动编排

1. 创建活动并绑定直播基础信息。
2. 选择发送策略（客户群/外部联系人/两者）。
3. 选择发送主体成员（真实 `userid`）。
4. 选择触发方式（立即/定时）。

### 4.2 小鹅通执行能力

1. 自动登录态校验。
2. 自动创建直播并返回 `live_id`。
3. 挂载商品与优惠券。
4. 输出直播链接与基础运营信息。

### 4.3 企业微信群发能力

1. 按成员身份发起客户群发任务。
2. 按成员身份发起外部联系人群发任务。
3. 记录企微任务 ID、状态、失败原因。

### 4.4 状态与可观测性

状态流转:
1. `draft`
2. `scheduled`
3. `running`
4. `partial_failed`
5. `success` / `failed`

## 5. 非功能需求

1. 幂等: 同一活动同一目标类型重复触发不重复发送。
2. 安全: 凭据加密存储，成员映射受控。
3. 审计: 记录发起人、执行时间、发送对象范围、发送结果。
4. 稳定性: 页面自动化步骤具备截图与失败快照。
5. 可运维: 错误分类清晰，支持自动重试和人工补发。

## 6. 验收标准（V1）

1. 直播创建成功率 >= 95%。
2. 群发任务提交成功率 >= 98%。
3. 单次活动从配置到执行不超过 5 分钟（不含定时等待）。
4. 失败任务具备可读错误原因且可重试。

## 7. 里程碑

1. M1（第 1 周）: 小鹅通创建直播 + 带货 + 优惠券打通。
2. M2（第 2 周）: 客户群群发打通（P0）。
3. M3（第 3 周）: 外部联系人群发打通（P1）。
4. M4（第 4 周）: 状态追踪、重试、灰度上线。

## 8. 未来阶段（V2+）

1. 引入小鹅通 API 适配器（与网页自动化双通道）。
2. 增加企业微信内部应用消息与群机器人能力。
3. 增加分层触达规则引擎与自动化运营剧本。
4. 建立触达到成交的数据归因闭环。
